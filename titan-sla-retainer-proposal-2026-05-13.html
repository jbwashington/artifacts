Decisions needed before this goes live

Open questions

  1. "Hours" definition. I'm proposing active API time (the duration the model is actually working) as the meter, because that's what the billing snapshot captures honestly. But clients may push for "wall-clock hours" or "session count." If we go with active time we need a clear explanation of what counts — happy to draft.
  2. Overage approval threshold. I picked "$30 estimated agent cost" as the line where overage work needs pre-approval (≈ $300 client-facing). Too low (friction), too high (surprise bills), about right?
  3. Quarterly review cadence. Should we build in a "review & reset" every quarter where retainer levels can flex based on actual usage trends? Or annual?
  4. What about future clients? Should these three retainers be one-offs, or do we standardize as named tiers (e.g., "Titan SLA — Pro / Standard / Light") that future clients pick from? Standardization is cleaner long-term but harder to fit existing client needs into.
  5. Annual prepay discount. Worth offering? E.g., 10% off if they prepay 12 months. Improves cashflow, locks them in. Or do we stay month-to-month for now?
